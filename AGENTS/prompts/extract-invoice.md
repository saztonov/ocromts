# Инструкция: Извлечение позиций из накладной/счёта поставщика

## Задача

Прочитай файл накладной/счёта поставщика и извлеки все позиции материалов в структурированный JSON.

---

## Режим 1: PDF-документ

### Типы документов
- ТОРГ-12 (товарная накладная)
- УПД (универсальный передаточный документ)
- Счёт на оплату
- Счёт-фактура
- Накладная на отпуск материалов

### Правила извлечения из PDF

1. **Наименование (rawName)**: Копируй ТОЧНО как написано в документе, включая все сокращения, ГОСТы, размеры, марки. Не исправляй опечатки, не переформатируй.

2. **Единица измерения (unit)**: Нормализуй (см. таблицу ниже).

3. **Количество (quantity)**: Число. Если указано с запятой как десятичный разделитель — преобразуй в точку.

4. **Цена за единицу (unitPrice)**: Число или null если отсутствует.

5. **Сумма (totalPrice)**: Число или null если отсутствует.

6. **Пропускай строки**:
   - Итого / Всего / В том числе НДС / Итого с НДС
   - Заголовки разделов
   - Пустые строки
   - Строки с услугами доставки (если явно не является материалом)

### Метаданные документа

Извлеки из шапки/подвала документа:
- `document_type` — тип документа (накладная, счёт-фактура, УПД, счёт на оплату)
- `supplier_name` — наименование поставщика (или null)
- `document_number` — номер документа (или null)
- `document_date` — дата документа (или null)

---

## Режим 2: Excel-документ

Парсинг аналогичен файлу заказа (см. `extract-order.md`), но с дополнительными колонками:

| Колонка | Ключевые слова |
|---------|---------------|
| **Наименование** | наименование, название, материал, товар, описание, номенклатура |
| **Ед. измерения** | ед, ед.изм, единица, изм, ед. изм., единица измерения |
| **Количество** | кол-во, количество, объём, объем, кол., кол, к-во |
| **Цена за ед.** | цена, цена за ед |
| **Сумма** | сумма, итого, стоимость, всего |

Правила пропуска строк — те же, что в `extract-order.md`.

Метаданные: попробуй определить тип документа, поставщика, номер и дату из строк перед таблицей.

## Нормализация единиц измерения

| Варианты | Нормализованная форма |
|----------|----------------------|
| м.п., п.м., пог.м, погонный метр | м.п. |
| м.кв, кв.м, м², м2, метр квадратный | м² |
| м.куб, куб.м, м³, м3, метр кубический | м³ |
| шт., штук, штука, штк. | шт |
| кг., килограмм | кг |
| т., тонна | т |
| м., метр | м |
| л., литр | л |
| компл., комплект | компл |
| уп., упаковка | уп |
| рул., рулон | рул |
| подд., поддон | подд |

## Формат выходного JSON

Сохрани результат в файл `output/invoice_items.json`:

```json
{
  "document_type": "товарная накладная",
  "supplier_name": "ООО \"СтальТрейд\"",
  "document_number": "СТ-2024/156",
  "document_date": "2024-03-15",
  "items": [
    {
      "position": 1,
      "rawName": "Труба стальная бесшовная 89х4 ГОСТ 8732-78 ст.20",
      "unit": "м.п.",
      "quantity": 150,
      "unitPrice": 450.00,
      "totalPrice": 67500.00
    },
    {
      "position": 2,
      "rawName": "Лист горячекатаный 4мм 1500х6000 Ст3сп ГОСТ 19903-2015",
      "unit": "т",
      "quantity": 2.5,
      "unitPrice": 65000.00,
      "totalPrice": 162500.00
    }
  ],
  "totals": {
    "subtotal": 230000.00,
    "vat": 46000.00,
    "total": 276000.00
  }
}
```

## Важно

- Извлекай ВСЕ строки материалов, даже если их много
- НЕ пропускай строки с длинными наименованиями
- НЕ объединяй похожие позиции
- Каждая строка таблицы = отдельный элемент в items
- `position` — порядковый номер строки начиная с 1 (или номер позиции из документа, если он указан)
- `rawName` — ТОЧНАЯ копия наименования, без исправлений
- Цены и суммы — числа с точкой (не запятой)
