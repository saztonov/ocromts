# Инструкция: Извлечение позиций из файла заказа (Excel)

## Задача

Прочитай Excel-файл заказа поставщику и извлеки все позиции материалов в структурированный JSON.

## Стратегия парсинга

### 1. Определение заголовков

Просканируй первые 20 строк файла. Ищи строку-заголовок по ключевым словам в ячейках:

| Колонка | Ключевые слова |
|---------|---------------|
| **Наименование** | наименование, название, материал, товар, описание, номенклатура |
| **Ед. измерения** | ед, ед.изм, единица, изм, ед. изм., единица измерения |
| **Количество** | кол-во, количество, объём, объем, кол., кол, к-во |
| **Цена** (опц.) | цена, цена за ед |
| **Сумма** (опц.) | сумма, итого, стоимость, всего |

**Минимальное требование:** найдены колонки "Наименование" + "Количество".

Если колонка "Ед. изм." не найдена — проверь колонку между "Наименование" и "Количество".

### 2. Чтение данных

Начни читать данные со строки после заголовка. Для каждой строки:
- Возьми значение из колонки "Наименование"
- Возьми значение из колонки "Количество" (должно быть положительное число)
- Возьми значение из колонки "Ед. изм." (если есть)

### 3. Правила пропуска строк

**Пропускай строки:**
- Пустые строки (нет наименования)
- Заголовки разделов (строка заканчивается на `:`)
- Итоговые строки: начинаются с "итого", "всего", "подитог", "в том числе", "НДС", "налог", "скидка", "наценка", "доставка", "total", "subtotal"
- Строки с "раздел " в начале
- Строки с нулевым или отрицательным количеством
- Строки где количество не является числом

### 4. Fallback (без заголовков)

Если заголовки не найдены, считай что колонки идут в порядке: [№, Наименование, Ед.изм., Количество, ...].
Ищи в каждой строке: текстовую ячейку длиной > 5 символов (наименование), короткую текстовую ячейку (ед.изм.), числовую ячейку > 0 (количество).

## Нормализация единиц измерения

Приведи единицы к стандартной форме:

| Варианты | Нормализованная форма |
|----------|----------------------|
| м.п., п.м., пог.м, погонный метр | м.п. |
| м.кв, кв.м, м², м2, метр квадратный | м² |
| м.куб, куб.м, м³, м3, метр кубический | м³ |
| шт., штук, штука, штк. | шт |
| кг., килограмм | кг |
| т., тонна | т |
| м., метр | м |
| л., литр | л |
| компл., комплект | компл |
| уп., упаковка | уп |
| рул., рулон | рул |
| подд., поддон | подд |

## Формат выходного JSON

Сохрани результат в файл `output/order_items.json`:

```json
[
  {
    "position": 1,
    "rawName": "Труба стальная б/ш 89х4 ГОСТ 8732-78 ст.20",
    "unit": "м.п.",
    "quantity": 150
  },
  {
    "position": 2,
    "rawName": "Лист г/к 4мм 1500х6000 Ст3сп ГОСТ 19903-2015",
    "unit": "т",
    "quantity": 2.5
  }
]
```

## Важно

- `rawName` — ТОЧНАЯ копия наименования из файла, без изменений
- `position` — порядковый номер (1, 2, 3...), начиная с первой валидной строки
- `quantity` — число с точкой как десятичным разделителем (если в файле запятая — замени на точку)
- Извлекай ВСЕ строки материалов, даже с длинными наименованиями
- НЕ объединяй похожие позиции
