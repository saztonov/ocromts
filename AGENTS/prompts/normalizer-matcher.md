# Инструкция: Нормализация наименований и сопоставление позиций

## Задача

Прочитай извлечённые данные из `output/order_items.json` и `output/invoice_items.json`, выполни нормализацию наименований, сопоставь позиции заказа с позициями накладной, проверь количества и классифицируй расхождения.

---

## Этап 2: Сопоставление позиций

Сопоставляй позиции заказа с позициями накладной по **всем извлечённым атрибутам**, в порядке приоритета:

### Приоритет атрибутов при сопоставлении

| Приоритет | Атрибут | Правило | Severity при несовпадении |
|-----------|---------|---------|--------------------------|
| 1 (обязат.) | Тип материала | Должен совпадать. Без совпадения типа — не сопоставлять | — (не сопоставляется) |
| 2 (критич.) | Ключевые размеры (D, s, B, H, t, сечение) | Допуск ±1мм для металлопроката, точное для остального | critical |
| 3 (критич.) | Марка / сорт / класс | Ст3 ≠ 09Г2С, А400 ≠ А500С, D400 ≠ D500 | critical |
| 4 (критич.) | Класс прочности | В15 ≠ В25, М200 ≠ М300, 8.8 ≠ 10.9 | critical |
| 5 (важный) | ГОСТ / ТУ / СТО | Если указан в обоих — должен совпадать | warning (если размеры ОК), critical (если ГОСТ меняет характеристики) |
| 6 (важный) | Способ изготовления | г/к ≠ х/к, бесшовная ≠ электросварная | warning |
| 7 (важный) | Плотность (D), теплопроводность (λ) | Должны совпадать для утеплителей, блоков | critical |
| 8 (важный) | Морозостойкость (F), водонепроницаемость (W) | Если указаны — должны совпадать | warning |
| 9 (важный) | Группа горючести | НГ ≠ Г1 ≠ Г4 | critical (для пожаробезопасности) |
| 10 (средний) | Давление (PN), SDR | Если указаны — должны совпадать | critical |
| 11 (средний) | Покрытие | оцинкованный ≠ без покрытия, RAL цвет | warning |
| 12 (средний) | Длина, формат | 6000 ≠ 12000, если критично для использования | warning |
| 13 (низкий) | Бренд / производитель | Другой бренд при тех же характеристиках | warning |
| 14 (низкий) | Фасовка, объём тары | 25кг ≠ 50кг (влияет только на количество упаковок) | info |
| 15 (инфо) | Форматирование, сокращения | "б/ш" = "бесшовная", "г/к" = "горячекатаный" | info |

### Допуски при сравнении размеров

| Категория | Допуск |
|-----------|--------|
| Металлопрокат (D, s, B, H, t) | ±1 мм |
| Арматура (Ø) | точное совпадение (Ø12 ≠ Ø14) |
| Трубы инженерные (D) | точное совпадение (Ø25 ≠ Ø32) |
| Крепёж (M) | точное совпадение (М10 ≠ М12) |
| Утеплитель (толщина) | точное совпадение (50мм ≠ 100мм) |
| Кирпич, блоки (LxBxH) | ±2 мм |
| Пиломатериалы (сечение) | ±2 мм |
| Длина (мерная) | ±50 мм |

### Общие правила
- Одна позиция заказа ↔ одна позиция накладной (строго 1:1)
- Если точного совпадения нет — ищи ближайшее с указанием ВСЕХ расхождений по атрибутам
- Не сопоставляй с confidence < 0.5 — лучше оставить в unmatched
- При расхождении — указывай в `parameter_mismatches` **каждый** отличающийся атрибут

### Уверенность (match_confidence)
| Диапазон | Описание |
|----------|----------|
| 1.0 | Идеальное совпадение всех параметров |
| 0.8–0.99 | Совпадает с мелкими различиями (форматирование, порядок слов) |
| 0.5–0.79 | Вероятное совпадение с расхождениями в параметрах |
| < 0.5 | Сомнительное — НЕ сопоставлять, оставить в unmatched |

---

## Этап 3: Проверка количеств

Сравни количества с учётом единиц измерения:

| Из | В | Формула |
|----|---|---------|
| кг | т | ÷ 1000 |
| т | кг | × 1000 |
| м² | 100м² | ÷ 100 |
| м.п. | шт | если известна длина штуки |
| м³ | м³ | прямое сравнение |

### Статусы количества
- **exact** — разница 0%
- **within_tolerance** — разница ≤ 5%
- **over** — в накладной больше, чем в заказе, разница > 5%
- **under** — в накладной меньше, чем в заказе, разница > 5%
- **incompatible_units** — единицы нельзя привести друг к другу без дополнительных данных

Для каждого сопоставления укажи:
- `conversion_note` — формула пересчёта (например "0.1 т = 100 кг")
- `difference_pct` — процент разницы (относительно заказа)

---

## Этап 4: Классификация расхождений

Для каждого сопоставленного элемента укажи все расхождения параметров:

### Severity уровни

**critical** — расхождение влияет на конструкцию, безопасность или функциональность:
- Размеры (D, s, B, H, t, сечение) за пределами допуска
- Марка стали / материала (Ст3 вместо 09Г2С)
- Класс прочности (В15 вместо В25, А400 вместо А500С, 8.8 вместо 10.9)
- Плотность утеплителя/блоков (D400 вместо D500)
- Группа горючести (Г4 вместо НГ)
- Давление (PN10 вместо PN16)
- Теплопроводность (λ 0.040 вместо 0.032)
- Диаметр арматуры, крепежа, труб

**warning** — расхождение значимое, но не критичное:
- Другой бренд/производитель при тех же характеристиках
- ГОСТ отличается, но параметры совпадают
- Способ изготовления (г/к vs х/к) при одинаковых размерах
- Покрытие отличается (оцинкованный vs без покрытия)
- Морозостойкость (F50 vs F75)
- Длина мерности (немерная vs мерная)
- Цвет (RAL)

**info** — различие не влияет на применение:
- Различия в форматировании, сокращениях, порядке слов
- Фасовка (25кг vs 50кг — влияет только на упаковку)
- Класс точности при одинаковых размерах

## Батчинг (при большом количестве позиций)

Если в заказе **больше 30 позиций**:
1. Раздели позиции заказа на батчи по 30
2. Для каждого батча:
   - Обработай только текущие 30 позиций заказа
   - Используй ВСЕ доступные позиции накладной
   - Запоминай, какие позиции накладной уже сопоставлены
3. В следующем батче:
   - Исключи из доступных те позиции накладной, которые уже сопоставлены
4. После всех батчей:
   - Собери все matched_items в один массив
   - Оставшиеся позиции заказа → unmatched_order
   - Оставшиеся позиции накладной → unmatched_invoice
