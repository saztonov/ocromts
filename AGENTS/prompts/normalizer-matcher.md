# Инструкция: Нормализация наименований и сопоставление позиций

## Задача

Прочитай извлечённые данные из `output/order_items.json` и `output/invoice_items.json`, выполни нормализацию наименований, сопоставь позиции заказа с позициями накладной, проверь количества и классифицируй расхождения.

---

## Этап 1: Нормализация наименований

Для каждой позиции из обоих списков выдели:
- **Тип материала** (труба, лист, уголок, швеллер, арматура, балка, профнастил, утеплитель, пиломатериал и т.д.)
- **Размеры**: диаметр, толщина стенки, ширина, высота, длина, толщина (в мм)
- **Марка стали / материала**: Ст3, 09Г2С, 3сп/пс, AISI 304 и т.д.
- **ГОСТ / ТУ**: ГОСТ 8732-78, ГОСТ 8240-97 и т.д.
- **Особенности**: покрытие (оцинкованная, грунтованная), обработка

### Расшифровка сокращений

| Сокращение | Полное наименование |
|-----------|---------------------|
| б/ш, бш | бесшовная |
| г/к, г.к., горячекат | горячекатаная |
| х/к, х.к., холоднокат | холоднокатаная |
| э/с, э.с., электросв | электросварная |
| оц, оцинк | оцинкованная |
| н/м, н.м. | немерной длины |
| м/д, мерн | мерной длины |
| ст. | сталь |
| s | толщина стенки |
| t | толщина |

При нормализации учитывай, что одна и та же позиция может быть записана по-разному:
- "Труба б/ш 89х4 ст.20" = "Труба стальная бесшовная 89х4 ГОСТ 8732-78 Сталь 20"
- "Лист г/к 4мм" = "Лист горячекатаный толщиной 4 мм"

---

## Этап 2: Сопоставление позиций

Сопоставляй позиции заказа с позициями накладной по:
1. **Тип материала** — должен совпадать
2. **Ключевые размеры** — диаметр, толщина стенки, ширина, высота (допуск ±1мм для проката)
3. **ГОСТ / ТУ** — если указан в обоих документах, должен совпадать
4. **Марка стали** — если указана

### Правила сопоставления
- Одна позиция заказа ↔ одна позиция накладной (строго 1:1)
- Если точного совпадения нет — ищи ближайшее с указанием расхождений
- Не сопоставляй с confidence < 0.5 — лучше оставить в unmatched

### Уверенность (match_confidence)
| Диапазон | Описание |
|----------|----------|
| 1.0 | Идеальное совпадение всех параметров |
| 0.8–0.99 | Совпадает с мелкими различиями (форматирование, порядок слов) |
| 0.5–0.79 | Вероятное совпадение с расхождениями в параметрах |
| < 0.5 | Сомнительное — НЕ сопоставлять, оставить в unmatched |

---

## Этап 3: Проверка количеств

Сравни количества с учётом единиц измерения:

| Из | В | Формула |
|----|---|---------|
| кг | т | ÷ 1000 |
| т | кг | × 1000 |
| м² | 100м² | ÷ 100 |
| м.п. | шт | если известна длина штуки |
| м³ | м³ | прямое сравнение |

### Статусы количества
- **exact** — разница 0%
- **within_tolerance** — разница ≤ 5%
- **over** — в накладной больше, чем в заказе, разница > 5%
- **under** — в накладной меньше, чем в заказе, разница > 5%
- **incompatible_units** — единицы нельзя привести друг к другу без дополнительных данных

Для каждого сопоставления укажи:
- `conversion_note` — формула пересчёта (например "0.1 т = 100 кг")
- `difference_pct` — процент разницы (относительно заказа)

---

## Этап 4: Классификация расхождений

Для каждого сопоставленного элемента укажи все расхождения параметров:

### Severity уровни
- **critical** — неправильные размеры, марка стали, тип материала (может повлиять на конструкцию)
- **warning** — другой бренд при тех же характеристиках, отличие ГОСТа при совпадении параметров
- **info** — различия в форматировании, сокращениях, порядке слов

---

## Батчинг (при большом количестве позиций)

Если в заказе **больше 30 позиций**:
1. Раздели позиции заказа на батчи по 30
2. Для каждого батча:
   - Обработай только текущие 30 позиций заказа
   - Используй ВСЕ доступные позиции накладной
   - Запоминай, какие позиции накладной уже сопоставлены
3. В следующем батче:
   - Исключи из доступных те позиции накладной, которые уже сопоставлены
4. После всех батчей:
   - Собери все matched_items в один массив
   - Оставшиеся позиции заказа → unmatched_order
   - Оставшиеся позиции накладной → unmatched_invoice

## Формат выходного JSON

Сохрани результат в файл `output/comparison.json`:

```json
{
  "metadata": {
    "order_file": "имя файла заказа",
    "invoice_file": "имя файла накладной",
    "invoice_type": "pdf или excel",
    "comparison_date": "YYYY-MM-DD",
    "folder": "AGENTS/compare/N"
  },
  "matched_items": [
    {
      "order_row": 1,
      "invoice_row": 3,
      "order_name": "исходное наименование из заказа",
      "invoice_name": "исходное наименование из накладной",
      "normalized_name": "нормализованное общее название",
      "match_confidence": 0.95,
      "match_reasoning": "пояснение почему сопоставлены",
      "parameter_mismatches": [
        {
          "parameter": "название параметра",
          "order_value": "значение в заказе",
          "invoice_value": "значение в накладной",
          "severity": "critical | warning | info"
        }
      ],
      "quantity_comparison": {
        "order_qty": 100,
        "order_unit": "кг",
        "invoice_qty": 0.1,
        "invoice_unit": "т",
        "converted_invoice_qty": 100,
        "converted_unit": "кг",
        "difference_pct": 0,
        "status": "exact",
        "conversion_note": "0.1 т = 100 кг"
      }
    }
  ],
  "unmatched_order": [
    {
      "order_row": 5,
      "order_name": "наименование из заказа",
      "reason": "причина почему не найдено совпадение"
    }
  ],
  "unmatched_invoice": [
    {
      "invoice_row": 7,
      "invoice_name": "наименование из накладной",
      "reason": "причина почему не найдено совпадение"
    }
  ],
  "summary": {
    "total_order": 10,
    "total_invoice": 12,
    "matched": 8,
    "unmatched_order": 2,
    "unmatched_invoice": 4,
    "critical_mismatches": 1,
    "warnings": 3
  }
}
```

## Важно

- **Полнота**: matched + unmatched_order = total_order; matched + unmatched_invoice = total_invoice
- **Без дублей**: каждая позиция накладной сопоставлена максимум с одной позицией заказа
- `order_name` и `invoice_name` — ТОЧНЫЕ копии из исходных файлов (rawName)
- `normalized_name` — общее нормализованное название для сопоставленной пары
- Для несопоставленных позиций укажи осмысленную причину (не просто "не найдено")
