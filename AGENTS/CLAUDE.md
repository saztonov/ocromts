# Пайплайн сверки заказов поставщику и накладных/счетов поставщика

## Описание проекта

Автоматизированный пайплайн сверки заказа поставщику (Excel) и накладной/счёта от поставщика (PDF или Excel).
Работает в Claude Code CLI через skills, MCP-серверы и субагентов.

## Структура данных

Каждая сверка — подпапка `compare/{N}/` (например `compare/1/`, `compare/2/`).

Пользователь размещает в папке:
- Файл **заказа** (`.xlsx` / `.xls`) — список заказанных материалов с количествами
- Файл **накладной/счёта** (`.pdf` / `.xlsx` / `.xls`) — документ от поставщика

Результаты сохраняются в `compare/{N}/output/`:
- `order_items.json` — извлечённые позиции заказа
- `invoice_items.json` — извлечённые позиции накладной
- `comparison.json` — результат сопоставления (машинный формат)
- `report.md` — читаемый отчёт (Markdown)
- `report.xlsx` — табличный отчёт (Excel, 3 листа)

## Поддерживаемые форматы

| Документ | Форматы | Примечание |
|----------|---------|------------|
| Заказ поставщику | `.xlsx`, `.xls` | Таблица с наименованиями, ед.изм., количествами |
| Накладная / счёт | `.pdf`, `.xlsx`, `.xls` | ТОРГ-12, УПД, счёт на оплату, счёт-фактура |

## Определение файлов

- `.pdf` — всегда накладная/счёт
- Если один `.xlsx` + один `.pdf` → xlsx = заказ, pdf = накладная
- Если несколько `.xlsx`: имя содержит "заказ"/"order" → заказ; "накладная"/"invoice"/"счёт" → накладная
- Если не удаётся определить → спросить пользователя

## MCP-серверы

### filesystem
Чтение/запись файлов в `AGENTS/`. Tools: `read_file`, `list_directory`, `write_file`.

### excel
Работа с Excel-файлами. Tools: `excel_describe_sheets`, `excel_read_range`.

## Пайплайн (5 фаз)

При вызове `/compare <N>` выполняются:

0. **Discovery** — сканирование папки, определение файлов заказа и накладной
1. **Extraction** — парсинг заказа (Excel) и накладной (PDF/Excel) в JSON
2. **Comparison** — нормализация наименований, сопоставление позиций, проверка количеств
3. **Validation** — проверка полноты: каждая позиция учтена, нет дублей
4. **Output** — генерация отчётов (MD + Excel), вывод сводки

## Команда агентов

| Агент | Роль | Задачи |
|-------|------|--------|
| `extractor` | Парсер документов | Читает файлы, извлекает позиции материалов |
| `comparator` | Сопоставитель | Нормализует наименования, сопоставляет позиции |
| `lead` | Координатор | Управляет процессом, валидирует, генерирует отчёты |

## Правила нормализации наименований

Полный каталог атрибутов по категориям — см. `AGENTS/prompts/normalizer-matcher.md`, раздел 1.2.

### Категории материалов с ключевыми атрибутами

| Категория | Ключевые атрибуты для сравнения |
|-----------|-------------------------------|
| Трубы стальные | D, s, тип (б/ш, э/с, ВГП), марка стали, ГОСТ |
| Листы и полосы | t, BxL, тип (г/к, х/к), марка стали, ГОСТ |
| Фасонный прокат | №/размер, h, b, s, t, тип, марка стали, ГОСТ |
| Арматура | Ø, класс (А400, А500С), профиль, ГОСТ |
| Профнастил | марка (С21, Н60), t металла, покрытие, цвет RAL |
| Бетон и ЖБИ | класс (В25), F, W, подвижность, марка ЖБИ, размеры |
| Кирпич и блоки | тип, M, F, D (плотность), LxBxH, пустотность |
| Утеплители | тип, плотность, толщина, λ, группа горючести |
| Гидроизоляция | тип, толщина, основа, марка, модификация (ЭПП/ЭКП) |
| Пиломатериалы | тип, порода, сечение BxH, длина, сорт, влажность |
| Сухие смеси | тип, марка/серия, основа, фасовка |
| Трубы инженерные | материал (ПП, ПВХ), D, s, PN, SDR |
| Кабельная продукция | марка, сечение, число жил, напряжение, огнестойкость |
| Крепёж | тип, M/Ø, длина, класс прочности, материал |

### Расшифровка основных сокращений
| Сокращение | Полное | Сокращение | Полное |
|-----------|--------|-----------|--------|
| б/ш | бесшовная | PN | номинальное давление |
| г/к | горячекатаная | DN | номинальный диаметр |
| х/к | холоднокатаная | SDR | размерное отношение |
| э/с | электросварная | НГ, Г1-Г4 | группа горючести |
| оц | оцинкованная | ВГП | водогазопроводная |
| н/м | немерной длины | XPS, EPS | экструд./вспен. пенополистирол |
| м/д | мерной длины | ФК, ФСФ | тип клея фанеры |
| ст. | сталь | нг(А)-LS | не распр. горение, низкое дымовыд. |
| s, t | толщина стенки / толщина | D, Ø | диаметр |

## Правила сопоставления

### Критерии совпадения
1. **Тип материала** — должен совпадать
2. **Ключевые размеры** — допуск ±1мм для проката
3. **ГОСТ / ТУ** — если указан в обоих документах
4. **Марка стали** — если указана

### Уверенность (match_confidence)
| Уровень | Значение | Описание |
|---------|----------|----------|
| Идеальное | 1.0 | Все параметры совпадают |
| Хорошее | 0.8–0.99 | Мелкие различия в форматировании |
| Вероятное | 0.5–0.79 | Есть расхождения в параметрах |
| Сомнительное | < 0.5 | Много несовпадений |

### Классификация расхождений (severity)
| Уровень | Примеры |
|---------|---------|
| **critical** | Неправильные размеры, марка стали, тип материала |
| **warning** | Другой бренд при тех же характеристиках, отличие ГОСТа |
| **info** | Различия в форматировании, сокращениях, порядке слов |

## Нормализация единиц измерения

### Таблица нормализации
| Варианты написания | Нормализованная форма |
|-------------------|----------------------|
| м.п., п.м., пог.м | м.п. |
| м.кв, кв.м, м², м2 | м² |
| м.куб, куб.м, м³, м3 | м³ |
| шт., штук, штука | шт |
| кг., килограмм | кг |
| т., тонна | т |
| компл., комплект | компл |
| уп., упаковка | уп |
| л., литр | л |
| рул., рулон | рул |
| подд., поддон | подд |
| 100 м², 100 м.кв. | 100м² |

### Конвертация при сравнении количеств
| Из | В | Формула |
|----|---|---------|
| кг | т | ÷ 1000 |
| т | кг | × 1000 |
| м² | 100м² | ÷ 100 |
| м.п. | шт | если известна длина штуки |

### Статусы количества
- **exact** — разница 0%
- **within_tolerance** — разница ≤ 5%
- **over** — в накладной больше (> 5%)
- **under** — в накладной меньше (> 5%)
- **incompatible_units** — единицы нельзя привести друг к другу

## Экспорт отчётов

### Markdown-отчёт
```bash
node AGENTS/scripts/to-markdown.js <папка>/output/comparison.json <папка>/output/report.md
```

### Excel-отчёт
```bash
node AGENTS/scripts/to-excel.js <папка>/output/comparison.json <папка>/output/report.xlsx
```

Excel создаёт 3 листа: Сопоставление, Несопоставленные, Сводка.

## Правила

- Каждая позиция заказа должна быть либо сопоставлена, либо явно помечена как несопоставленная
- Каждая позиция накладной должна быть либо сопоставлена, либо явно помечена как несопоставленная
- Одна позиция заказа ↔ одна позиция накладной (1:1)
- Наименования копируются ТОЧНО из документа (без исправления опечаток в raw_name)
- Нормализация — только для целей сопоставления (normalized_name)
- Промежуточные данные сохранять в `<папка>/output/`
- При > 30 позициях заказа — обрабатывать батчами по 30
