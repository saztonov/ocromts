# Сверка заказа и накладной поставщика

Тебе передан номер папки `$ARGUMENTS`. Рабочая папка: `AGENTS/compare/$ARGUMENTS/`.

## Шаг 0: Проверка MCP-серверов

**ОБЯЗАТЕЛЬНО выполни перед созданием команды.**

Запусти 2 проверки **параллельно** (одним сообщением с 2 tool calls):

```
1. mcp__filesystem__list_directory(path="compare/$ARGUMENTS")
2. mcp__excel__excel_describe_sheets(fileAbsolutePath="<любой_xlsx_из_папки>") ИЛИ пропустить если xlsx не найден
```

**Оценка результатов:**

| Сервер | Если OK | Если ОШИБКА |
|--------|---------|-------------|
| filesystem | Продолжай | СТОП. Без filesystem невозможно читать файлы. Проверьте settings.json |
| excel | Продолжай | Предупреди: "Excel MCP недоступен, экспорт через node scripts" |

**Константы для агентов:**
- `WORK_DIR = "AGENTS/compare/$ARGUMENTS"` (относительно корня проекта)
- `SCRIPTS_PATH = "AGENTS/scripts"` (относительно корня проекта)

Если MCP-проверки прошли — переходи к Шагу 1.

## Шаг 1: Обнаружение файлов (Discovery)

1. Просканируй папку `AGENTS/compare/$ARGUMENTS/` (Glob/Read)
2. Определи файл **заказа** и **накладной** по правилам:
   - `.pdf` — всегда накладная/счёт
   - Если несколько `.xlsx`: имя содержит "заказ"/"order" → заказ; "накладная"/"invoice"/"счёт" → накладная
   - Если один `.xlsx` + один `.pdf` → xlsx = заказ, pdf = накладная
   - Если не удаётся определить — **спроси пользователя**
3. Выведи таблицу найденных файлов, запроси подтверждение:

```
| Роль      | Файл              | Формат |
|-----------|--------------------|--------|
| Заказ     | order.xlsx         | Excel  |
| Накладная | invoice.pdf        | PDF    |
```

## Шаг 2: Создание команды агентов

Создай Team через TeamCreate:

```
team_name: "compare-$ARGUMENTS"
description: "Сверка заказа и накладной из папки compare/$ARGUMENTS"
```

Затем запусти 3 агентов через Agent tool с `team_name`:

### Агент 1: `extractor`
**Роль:** Парсер документов
**subagent_type:** general-purpose

Промпт:
```
Ты — extractor в команде compare. Твоя роль — извлечение данных из файлов заказа и накладной.

Рабочая папка: {ПОЛНЫЙ_ПУТЬ_К_ПАПКЕ}
Файл заказа: {ИМЯ_ФАЙЛА_ЗАКАЗА} (формат: {xlsx/xls})
Файл накладной: {ИМЯ_ФАЙЛА_НАКЛАДНОЙ} (формат: {pdf/xlsx})

Прочитай файлы промптов для детальных инструкций:
- AGENTS/prompts/extract-order.md — для парсинга заказа
- AGENTS/prompts/extract-invoice.md — для парсинга накладной

Твои обязанности:
1. Прочитать файл заказа (Read tool) и извлечь все позиции материалов
2. Прочитать файл накладной (Read tool) и извлечь все позиции материалов
3. Сохранить результаты:
   - {ПАПКА}/output/order_items.json — позиции заказа
   - {ПАПКА}/output/invoice_items.json — позиции накладной
4. Отправить результат агенту lead через SendMessage:
   "Извлечение завершено. Заказ: {N} позиций. Накладная: {M} позиций."

ВАЖНО:
- Копируй наименования ТОЧНО как написано в документе
- Нормализуй единицы измерения (м.п., м², м³, шт, кг, т, компл, уп)
- Пропускай итоговые строки, заголовки разделов, пустые строки
- Каждая строка таблицы = отдельный элемент

Начни с:
- Создай папку output если её нет (Bash: mkdir -p)
- Прочитай и обработай оба файла
- Сохрани JSON-файлы
- Отправь результаты lead
```

### Агент 2: `comparator`
**Роль:** Специалист по нормализации и сопоставлению
**subagent_type:** general-purpose

Промпт:
```
Ты — comparator в команде compare. Твоя роль — нормализация наименований и сопоставление позиций.

Рабочая папка: {ПОЛНЫЙ_ПУТЬ_К_ПАПКЕ}

Прочитай файл промпта для детальных инструкций:
- AGENTS/prompts/normalizer-matcher.md — правила нормализации, сопоставления, проверки количеств

Твои обязанности:
1. Прочитать output/order_items.json и output/invoice_items.json
2. Нормализовать наименования (расшифровка сокращений, стандартизация размеров)
3. Сопоставить позиции заказа с позициями накладной (1:1)
4. Проверить количества с учётом конвертации единиц (кг↔т, м²↔100м²)
5. Классифицировать расхождения (critical/warning/info)
6. Если позиций заказа > 30 — обрабатывать батчами по 30
7. Сохранить результат в output/comparison.json
8. Отправить результат агенту lead через SendMessage:
   "Сопоставление завершено. Совпало: {N}, не найдено в накладной: {M}, лишнее в накладной: {K}."

ВАЖНО:
- Каждая позиция заказа должна быть либо сопоставлена, либо в unmatched_order
- Каждая позиция накладной должна быть либо сопоставлена, либо в unmatched_invoice
- match_confidence: 1.0 = идеально, 0.8+ = мелкие различия, 0.5-0.8 = с расхождениями
- severity: critical = неправильные размеры/марка/тип, warning = другой бренд, info = форматирование

Начни с:
- Жди данные от extractor (или прочитай JSON если файлы уже есть)
- Выполни нормализацию и сопоставление
- Сохрани comparison.json
- Отправь результаты lead
```

### Агент 3: `lead` (team lead)
**Роль:** Координатор
**subagent_type:** general-purpose

Промпт:
```
Ты — lead (team lead) в команде compare. Ты координируешь сверку заказа и накладной.

Рабочая папка: {ПОЛНЫЙ_ПУТЬ_К_ПАПКЕ}
SCRIPTS_PATH: AGENTS/scripts

Команда:
- extractor — парсит файлы заказа и накладной
- comparator — нормализует и сопоставляет позиции
- ты (lead) — координируешь, валидируешь, генерируешь отчёты

Алгоритм работы:

1. ДОЖДИСЬ ИЗВЛЕЧЕНИЯ от extractor
   Жди сообщение о завершении извлечения.
   Проверь что файлы output/order_items.json и output/invoice_items.json существуют.

2. ЗАПУСТИ СОПОСТАВЛЕНИЕ
   SendMessage(to="comparator"): "Данные готовы. Начинай сопоставление."

3. ДОЖДИСЬ СОПОСТАВЛЕНИЯ от comparator
   Жди сообщение о завершении сопоставления.
   Проверь что файл output/comparison.json существует.

4. ВАЛИДАЦИЯ
   Прочитай comparison.json и проверь:
   - Все позиции заказа учтены (matched + unmatched_order = total_order)
   - Все позиции накладной учтены (matched + unmatched_invoice = total_invoice)
   - Нет дублирования (одна позиция накладной не сопоставлена с несколькими заказами)
   Если есть проблемы — отправь comparator на доработку.

5. ГЕНЕРАЦИЯ ОТЧЁТОВ
   Выполни последовательно:
   а) node AGENTS/scripts/to-markdown.js {ПАПКА}/output/comparison.json {ПАПКА}/output/report.md
   б) node AGENTS/scripts/to-excel.js {ПАПКА}/output/comparison.json {ПАПКА}/output/report.xlsx

6. ВЫВОД СВОДКИ ПОЛЬЗОВАТЕЛЮ
   Покажи:
   - Всего позиций заказа: N
   - Всего позиций накладной: M
   - Сопоставлено: K
   - Не найдено в накладной: X
   - Лишнее в накладной: Y
   - Критичных расхождений: Z
   - Предупреждений: W
   - Файлы: output/comparison.json, output/report.md, output/report.xlsx

7. ЗАВЕРШЕНИЕ
   SendMessage(to="extractor", message={type: "shutdown_request"})
   SendMessage(to="comparator", message={type: "shutdown_request"})

Начни с:
- Создай задачи в TaskList для команды
- Назначь extractor задачу извлечения данных
```

## Шаг 3: Создание задач

После создания команды создай задачи через TaskCreate:

1. `[extractor] Извлечь позиции из файла заказа (Excel)`
2. `[extractor] Извлечь позиции из файла накладной (PDF/Excel)`
3. `[comparator] Нормализовать и сопоставить позиции` (blocked by #1, #2)
4. `[lead] Валидация результатов сопоставления` (blocked by #3)
5. `[lead] Генерация отчётов (MD + Excel)` (blocked by #4)

## Интерактивный режим

Если пользователь не вызывает `/compare`, а работает через чат — создавай Team с теми же агентами, но без автоматических задач. Агенты ждут команд от пользователя:

```
Пользователь: "Извлеки данные из заказа"
→ назначь задачу extractor

Пользователь: "Сопоставь позиции"
→ назначь задачу comparator

Пользователь: "Сформируй отчёт"
→ назначь задачу lead
```
